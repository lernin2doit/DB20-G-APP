## Resolved Issues

### 1. HS_DN (Net 31) — Intentionally Spare ✅
- **Status:** Resolved (no action needed)
- **Details:** Declared but assigned to no component pads. The DB20-G RJ-45 handset interface uses 8 pins (HS_VCC, RADIO_MIC, PTT, GND×2, RADIO_SPK, HS_SPKM, HS_UP) which accounts for all 8 pins of the 8P8C connector. HS_DN is a spare net reserved for future use if a handset mic-down signal is needed.

### 2. USB Differential Pair Impedance — Documented ✅
- **Status:** Resolved (acceptable for prototype)
- **Details:** USB D+/D- traces use 0.3 mm width on a 2-layer 1.6 mm FR4 stackup. Calculated Zdiff ≈ 145Ω (target is 90Ω for USB 2.0). However:
  - CP2102N and CM108AH use Full Speed (12 Mbps) where impedance matching is non-critical.
  - FE1.1s upstream runs High Speed (480 Mbps) but trace lengths are <25 mm (well under λ/10), so reflections are negligible.
  - For production: increase trace width to ~0.65 mm with 0.15 mm gap, or move to a 4-layer stackup.
- **Notes documented in `generate_pcb.py` trace section.**

### 3. Footprint Courtyard/Fab Layers — Added ✅
- **Status:** Fixed
- **Details:** All footprints now include `F.Fab` body outline and `F.CrtYd` courtyard rectangles via the `pkg=` parameter in `generate_pcb.py`. Package dimensions defined per IPC-7351B:
  - 0805: 2.0×1.25 mm body, 3.4×1.9 mm courtyard
  - 1206: 3.2×1.6 mm body, 4.0×2.4 mm courtyard
  - SOD-123, SOT-223, SSOP-28, QFN-28, TO-92, HC49, USB-C, RJ-45, Audio Jack, Relay
- KiCad DRC courtyard overlap checks will now function correctly.

### 4. IC Pin Mapping — Verified ✅
- **Status:** Verified against datasheets
- **FE1.1s (U1, SSOP-28):** All 28 pins verified. DP0/DM0 → HUB_P1, DP1/DM1 → HUB_P2, XOUT(12)/XIN(13) → XO/XI, DP_UP(15)/DM_UP(16) → USB_D+/D-, VDD5/VDD33/AVDD33/AVDD power, TEST/SELF_PWR/GANG → GND, unused ports 2-3 → NC.
- **CP2102N-A02-GQFN28 (U2):** All 29 pins verified (28 + EP). D+/D-(4,5) → HUB_P1, TX/RX(19,20) → UART, /RTS/CTS/DTR(21-23), TX_LED(26), power on 6-9, EP(29) → GND.
- **CM108AH (U3, SSOP-28):** All 28 pins verified. USB_DP/DM(1,2) → HUB_P2, MIC_IN(15) → CM108_MIC, SPK_L/R(17,18) → CM108_SPK, GPIO4(13), VREF(16), VDD(6)/AVDD(19) → +5V.
- **Note:** CM108AH pin 7 (DVDD33) is NC. Datasheet recommends a 4.7–10 µF bypass cap from DVDD33 to GND for the internal 3.3V regulator. The chip functions without it, but adding one (C15) would improve digital supply filtering in a Rev 2.

### 5. Net 0 (Unconnected) Pads — Expected ✅
- **Status:** Resolved (by design)
- **Details:** Multiple IC pins are assigned to net 0 (""). These are intentionally unconnected (NC) pins:
  - U1: Pins 6-9 (unused hub ports 2-3), 19 (PDN_SEL), 22-25 (PWREN), 27 (REF)
  - U2: Pins 1-2 (DCD/RI), 10-18 (unused GPIOs), 24-25 (DSR/DCD), 27-28 (RX_LED/NC)
  - U3: Pins 4-5 (XIN/XOUT, internal osc), 7 (DVDD33), 9-12 (GPIO0-3), 14 (MIC_BIAS), 21-28 (I2S/I2C/NC)
- Net 0 is KiCad's standard designation for "no net." This is correct behavior.

### 5. Net 0 (Unconnected) Pads
- **Severity:** Info
- **Details:** Several IC pins are assigned to net 0 (unconnected). These are intentionally unused pins (e.g., FE1.1s ports 3/4, CM108AH I2S/I2C, CP2102N GPIO). In KiCad, these should have no-connect flags in the schematic.

## Remaining Steps Before Fabrication

1. **Open in KiCad** → "Update PCB from Schematic" to sync net assignments and add library footprint graphics
2. **Run DRC** in KiCad (Design Rule Check) to catch clearance violations, unconnected pads, and overlap issues
3. **Verify IC footprints** match ordered parts (especially QFN-28 thermal pad and SSOP-28 pin pitch)
4. **Generate Gerber files** using KiCad's Plot function with the `gerber/` output directory
5. **Generate drill file** (Excellon format) for PCB fabrication
6. **Upload to JLCPCB/PCBWay** for quote and fabrication
