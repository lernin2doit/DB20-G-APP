# KiCad Schematic Changes — v9 (USB) → v10 (Bluetooth ESP32)

> **Schematic file:** `hardware/kicad/DB20G-Interface-v10.kicad_sch`  
> **Generated by:** `generate_v10.py` (KiCad 9.0 S-expression format)  
> **ERC result:** 0 errors, 42 warnings (41 lib_symbol + 1 cosmetic)

## Summary

The v10 schematic replaces the entire USB signal chain (hub + UART bridge +
audio codec + crystal) with a single ESP32-WROOM-32E module.  The phone now
connects over Bluetooth instead of USB.  Component count dropped from ~55 to
~30.  Board cost dropped from ~$5.91 to ~$5.28.

---

## ICs Removed

| Ref | Part       | Package | Function                       | Why Removed |
|-----|------------|---------|--------------------------------|-------------|
| U1  | FE1.1s     | SSOP-28 | USB 2.0 4-port hub controller  | No USB to phone; ESP32 handles serial + audio natively |
| U2  | CP2102N    | QFN-28  | USB-to-UART bridge             | ESP32 has native UART; no USB path needed |
| U3  | CM108AH    | SSOP-28 | USB audio codec                | ESP32 ADC/DAC handles audio; BT audio to phone |
| Y1  | 12 MHz     | HC-49S  | Crystal for FE1.1s             | ESP32 has 40 MHz crystal on-module |

## Net Changes

### Nets Removed
All nets specific to the USB ICs were removed:
- `VD18OUT`, `VD33OUT`, `XI`, `XO`, `FE_RESET` (FE1.1s)
- `HUB_P1_DM`, `HUB_P1_DP`, `HUB_P2_DM`, `HUB_P2_DP` (hub downstream)
- `CP_TXD`, `CP_RXD`, `CP_TXLED`, `CTS_PU`, `PTT_CTRL`¹, `DTR_CTRL`¹ (CP2102N)
- `CM_VDD18`, `CM_VREF`, `CM_GPIO4`, `CM_RESET` (CM108AH)
- `AUDIO_OUT`, `AUDIO_RX`, `AUDIO_RX_DIV`, `AUDIO_TX_DIV` (CM108AH audio)
- `USB_DP`, `USB_DM`, `VBUS`, `CC1`, `CC2` (USB-C)

¹ Replaced by new ESP32 nets (see below).

## Relay K1 Wiring Changes

The relay function stays the same (switch MIC/SPK between serial and audio mode)
but the connected ICs change:

| Terminal | v9 Connection | v10 Connection |
|----------|---------------|----------------|
| Pole 1 COM | J2 pin 2 (MIC) | J2 pin 2 (MIC) — unchanged |
| Pole 1 NC | CM108AH audio out | ESP32 DAC (GPIO25) via divider |
| Pole 1 NO | CP2102N TXD | ESP32 UART2 TX (GPIO17) |
| Pole 2 COM | J2 pin 5 (SPK) | J2 pin 5 (SPK) — unchanged |
| Pole 2 NC | CM108AH audio in | ESP32 ADC (GPIO36) via divider |
| Pole 2 NO | CP2102N RXD | ESP32 UART2 RX (GPIO16) |
| Coil drive | CP2102N DTR → Q2 | ESP32 GPIO5 → Q2 |

## Power Design Changes

| Aspect | v9 | v10 |
|--------|-----|-----|
| Source | Phone VBUS (5 V via USB-C) | External 5 V on J4 or radio handset pin 1 |
| Regulator | AMS1117-3.3 (U4) | AMS1117-3.3 (U2, renumbered) |
| Protection | F1 polyfuse (500 mA) | F1 polyfuse (500 mA) — unchanged |
| Peak current | ~200 mA (3 ICs + relay) | ~350 mA (ESP32 BT TX + relay) |
| Phone power | Drained by board via OTG | Free — phone charges independently |
